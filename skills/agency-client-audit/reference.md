Also capture (product-marketing framing):

## Buyer Personas (B2B)

For each relevant stakeholder, fill the persona table in [templates.md](templates.md):

- User / Champion / Decision Maker / Technical Influencer / Financial Buyer
- What each cares about, their challenge, and the value the **client company** promises them

## Switching Dynamics (JTBD Four Forces)

Why would this client invest in a website redesign or agency engagement?

- **Push** — frustrations with current website/digital presence
- **Pull** — what attracts them to modernization (leads, credibility, export growth)
- **Habit** — what keeps them stuck (referrals work, "website is fine", family decision inertia)
- **Anxiety** — worries about switching (cost, downtime, losing SEO, vendor trust)

## Anti-Persona

Who is **NOT** a good fit for an agency pitch to this client? (e.g., purely commodity buyer, no budget authority, satisfied with current agency)

Label inferred insights vs. sourced facts. Capture **verbatim customer language** where found (website copy, LinkedIn, reviews, IndiaMART).

# Rules

- Use live web research wherever possible.
- Never invent facts.
- Prefer official sources over third-party sources.
- When estimating metrics (revenue, TAM, pricing), clearly label them as estimates.
- Prioritize actionable insights over generic observations.
- Support claims with links where applicable.
- Present findings in clean Markdown with headings, tables, and bullet points for readability.
- Think like a senior consultant preparing a client-ready discovery report that can directly feed into a proposal and sales conversation.

# STEP 17 — Export Report

After completing Steps 1–16, export the full report as markdown.

## 1. Ask output directory (required every run)

Use **AskQuestion** — do not assume a path silently. Offer:

- `reports/` (project default)
- Workspace root
- Custom path (user types path)

## 2. Filename convention

Sanitize company name to slug: trim, strip special characters, spaces → hyphens, collapse repeated hyphens.

| Company | Filename |
|---------|----------|
| Heli Fan | `Heli-Fan-Audit.md` |
| Golden Engineering | `Golden-Engineering-Audit.md` |
| Lorem Corp | `Lorem-Corp-Audit.md` |

## 3. Write markdown

Save the complete report to `{outputDir}/{CompanySlug}-Audit.md`.

**Required heading format** (for consistent report structure):

```markdown
# [Company Name] — Agency Research Report
*Prepared: [Month Year]*

# STEP 1 — Company Discovery
...
# STEP 2 — Business Understanding
...
```

Each `# STEP N — ...` heading must use this exact pattern through Step 16.

## 4. Confirm to user

Share the absolute path to the `.md` file.
